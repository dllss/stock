### 2. **新增字段（每个股票7个字段）**

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `name` | string | 股票名称 | "贵州茅台" |

| `market_name` | string | 交易所名称 | "上海证券交易所" |
| `market_code` | string | 交易所代码 | "SH", "SZ", "BJ" |
| `stock_type` | string | 板块类型 | "主板", "科创板", "创业板", "北交所" |
| `industry` | string | 所属行业 | "白酒", "半导体" |
| `listing_date` | string | 上市日期 | "20010827" |

## 📝 使用说明

### 1. 首次运行（生成新缓存）
```bash
python instock/job/historical_data_job.py --stock_code 000001 --start_date 2023-05-05 --end_date 2026-05-05
```

首次运行时会自动：
- 从API获取全市场股票信息
- 构造增强的对象结构
- 保存到 `instock/cache/stock_code_map.json`
- 显示详细的统计信息

### 2. 后续运行（使用缓存）
第二次及以后的运行会直接从缓存读取，速度非常快（毫秒级）。

### 3. 强制刷新缓存
```python
from instock.core.crawling.stock_hist_em import code_id_map_em
code_map = code_id_map_em(use_cache=False)  # 强制重新获取
```

## 🔧 技术细节

### API字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| f12 | 股票代码 | "600519" |
| f14 | 股票名称 | "贵州茅台" |
| f45 | 所处行业 | "白酒" |
| f57 | 上市时间 | "20010827" |

### 股票类型判断规则

- **科创板**: 代码以 `688` 或 `689` 开头
- **创业板**: 代码以 `300` 或 `301` 开头
- **北交所**: 市场ID为2
- **主板**: 其他情况

### 市场ID映射

- **0**: 深圳证券交易所 (SZ)
- **1**: 上海证券交易所 (SH)
- **2**: 北京证券交易所 (BJ)
