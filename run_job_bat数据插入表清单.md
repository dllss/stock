# run_job.bat 数据插入表完整清单

## 文档说明

本文档详细记录了运行 `run_job.bat`（即执行 `execute_daily_job.py`）会往哪些数据库表插入数据。

**数据来源**: 分析所有 job 目录下的任务脚本  
**生成时间**: 2026-05-03  
**总表数量**: **28个表**

---

## 执行流程概览

```
run_job.bat
    ↓
execute_daily_job.py (主调度器)
    ↓
┌─────────────────────────────────────────────┐
│ 第1步: init_job.py                          │
│ 初始化数据库（创建表结构）                    │
└────────────────┬────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│ 第2.1步: basic_data_daily_job.py            │
│ 抓取基础行情数据                              │
└────────────────┬────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│ 第2.2步: selection_data_daily_job.py        │
│ 抓取综合选股数据                              │
└────────────────┬────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│ 第3步: 并行执行（多线程）                     │
│ ├─ basic_data_other_daily_job.py            │
│ ├─ indicators_data_daily_job.py (已注释)     │
│ ├─ klinepattern_data_daily_job.py (已注释)   │
│ └─ strategy_data_daily_job.py (已注释)       │
└────────────────┬────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│ 第4步: backtest_data_daily_job.py (已注释)   │
│ 回测验证                                      │
└────────────────┬────────────────────────────┘
                 ↓
┌─────────────────────────────────────────────┐
│ 第5步: basic_data_after_close_daily_job.py  │
│ 收盘后数据                                    │
└─────────────────────────────────────────────┘
```

### ✅ 第2.1步：basic_data_daily_job.py

**插入的表：**

#### 1. cn_stock_spot - 每日股票数据

- **数据量**: 约5000+条/天（全部A股）

#### 2. cn_etf_spot - 每日ETF数据

- **更新频率**: 每个交易日
- **数据量**: 约500+条/天（全部ETF）

### ✅ 第2.2步：selection_data_daily_job.py

**插入的表：**

#### 3. cn_stock_selection - 综合选股数据
- **用途**: 存储东方财富综合选股的200+个指标
- **字段数**: **186个字段**（最宽的表）
- **数据来源**: 东方财富网综合选股API
- **数据内容**:
  - **基础信息**（13个字段）: 代码、名称、价格、涨跌幅等

  - **市值指标**（2个字段）: 总市值、流通市值
  - **每股指标**（8个字段）: EPS、BPS、每股现金流等
  - **利润指标**（4个字段）: 净利润、扣非净利润、营收
  - **盈利能力**（3个字段）: ROE、ROA、ROIC
  - **利润率**（2个字段）: 毛利率、净利率
  - **成长能力**（10个字段）: 净利润增长率、营收增长率等
  - **偿债能力**（5个字段）: 资产负债率、流动比率等
  - **股本结构**（2个字段）: 总股本、流通股本
  - **股东户数**（6个字段）: 股东人数、户均持股等
  - **股东持股**（2个字段）: 十大股东持股比例
  - **技术信号-MACD**（3个字段）: MACD金叉日/周/月线
  - **技术信号-KDJ**（3个字段）: KDJ金叉日/周/月线
  - **技术信号-突破**（8个字段）: 均线突破、多头排列等
  - **技术信号-量价**（7个字段）: 放量上涨、大阳线等
  - **技术信号-连续形态**（6个字段）: 连涨天数、连跌天数等
  - **技术信号-K线形态**（9个字段）: 吞没、锤头、十字星等
  - **解禁增发**（12个字段）: 限售解禁、定向增发
  - **质押商誉**（7个字段）: 股权质押、商誉规模
  - **业绩预告分红**（4个字段）: 业绩预告、分红方案
  - **高管机构**（4个字段）: 高管变动、机构调研
  - **机构持股-家数**（7个字段）: 基金、券商、QFII等
  - **机构持股-比例**（7个字段）: 各类机构持股比例
  - **人气排名**（10个字段）: 股吧人气、关注度排名
  - **高低点振幅**（15个字段）: 创新高/新低、破发破净
  - **跑赢大盘**（5个字段）: 近期跑赢大盘天数
  - **资金流向**（12个字段）: 净流入、DDX、增仓占比
  - **涨跌幅统计**（6个字段）: 多周期涨跌幅、连涨连跌
  - **其他指标**（4个字段）: 年化收益率、波动率等
- **更新频率**: 每个交易日
- **数据量**: 约5000+条/天
- **特点**: 数据最全、字段最多、抓取耗时最长（约5分钟）

## 注意事项

### 1. 网络要求
- 需要稳定的网络连接
- 大量API请求（东方财富网）
- 建议使用代理IP避免被封
- API可能有限流机制

### 2. 数据库要求
- MySQL服务必须正常运行
- 足够的存储空间（建议预留50GB+）
- 合理的索引设计
- 定期优化和备份

### 3. 执行时机
- **建议时间**: 交易日17:30后（收盘后）
- **避免时段**: 交易时段（9:30-15:00）
- **原因**: 
  - 收盘后数据最完整
  - 避免占用系统资源影响交易
  - API数据更稳定

### 4. 资源消耗
- **CPU**: 中等（并行任务时较高）
- **内存**: 500MB-1GB
- **网络**: 大量HTTP请求
- **磁盘**: 每日约18MB写入

### 5. 错误处理
- 某个任务失败不影响其他任务
- 查看日志文件定位问题
- 可以单独重新执行失败的任务
- 支持断点续传（指定日期）

### 6. 日志文件
- **位置**: `instock/log/stock_execute_job.log`
- **内容**: 任务开始/结束时间、错误信息、耗时统计
- **建议**: 定期清理或轮转日志文件

## 常见问题

### Q1: 为什么有些任务被注释了？
**A**: 可能是为了：
- 减少执行时间
- 降低资源消耗
- 按需启用功能
- 某些功能还在开发中

### Q2: 如何查看哪些表有数据？
**A**: 
```sql
-- 查看所有表的记录数
SELECT table_name, table_rows 
FROM information_schema.tables 
WHERE table_schema = 'your_database_name'
ORDER BY table_rows DESC;
```

### Q3: 数据更新失败怎么办？
**A**: 
1. 查看日志文件 `instock/log/stock_execute_job.log`
2. 检查网络连接
3. 检查MySQL服务是否正常
4. 检查是否有足够的磁盘空间
5. 尝试单独重新执行失败的任务

### Q4: 可以只更新部分表吗？
**A**: 可以，直接运行对应的任务脚本：
```bash
python instock/job/basic_data_daily_job.py  # 只更新基础数据

```

### Q5: 历史数据怎么补全？

# 补全单个日期
python instock/job/execute_daily_job.py 2024-01-01

python instock/job/execute_daily_job.py 2024-01-01,2024-01-02,2024-01-03

python instock/job/execute_daily_job.py 2024-01-01 2024-01-31
